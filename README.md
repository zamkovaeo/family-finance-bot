# Family Finance Telegram Bot

MVP 1.0 Telegram-бота для учета семейных финансов по PRD: расходы, доходы, категории, лимиты, личные/семейные операции, теги проектов, цели, отчеты и аналитика.

## Что умеет бот

- Главное меню Telegram: расход, доход, бюджет, семья, отчеты, цели, настройки.
- Свободный ввод: `Кофе 350 #отпуск личное 07.06.2026`, `Зарплата 180000 вчера`.
- Автокатегоризация по правилам с точкой расширения под LLM.
- Голосовой ввод через OpenAI Speech-to-Text при наличии `OPENAI_API_KEY`.
- Лимиты по категориям на месяц и уведомления на 80%, 100% и превышение.
- Теги проектов: ремонт, отпуск, автомобиль и другие через `#тег`.
- Финансовые цели с прогресс-барами.
- Отчеты: день, неделя, месяц.
- Графики: круговая диаграмма категорий, линия расходов по дням, составные столбцы месяцев.
- FastAPI endpoints: `/expense`, `/income`, `/budget`, `/analytics`, `/goals`, `/reports`.

## Быстрый запуск

1. Создайте Telegram-бота через `@BotFather` и получите токен.
2. Скопируйте настройки:

```bash
cp .env.example .env
```

3. Заполните в `.env`:

```env
TELEGRAM_BOT_TOKEN=ваш-токен
OPENAI_API_KEY=необязательно-для-голоса
```

4. Запустите:

```bash
docker compose up --build
```

5. Откройте API:

```text
http://localhost:8000/docs
```

## Локальный запуск без Docker

```bash
python -m venv .venv
.venv\Scripts\activate
pip install -r requirements.txt
alembic upgrade head
python -m app
```

Для локального запуска нужен PostgreSQL и корректный `DATABASE_URL` в `.env`.

## Быстрый деплой

Самый простой путь без локального Docker:

1. Загрузите проект в GitHub.
2. В Render создайте новый Blueprint из репозитория.
3. Render прочитает `render.yaml`, создаст Web Service и PostgreSQL.
4. В переменных окружения задайте `TELEGRAM_BOT_TOKEN`.
5. Если нужен голосовой ввод, добавьте `OPENAI_API_KEY`.

Строка PostgreSQL от Render автоматически приводится к формату `postgresql+asyncpg://`.

## Как пользоваться в Telegram

- `/start` создает семью и администратора.
- Код семьи показывается в настройках.
- Новый участник присоединяется командой `/start КОД`.
- Расход можно вводить сразу текстом: `Такси 780 #автомобиль 07.06.2026`.
- Личная операция: `Кофе 350 личное`.
- Дату можно указать вручную: `07.06.2026`, `07/06/2026`, `2026-06-07`, `вчера`, `позавчера`.
- Лимит: Настройки → Задать лимит → `Продукты 60000`.
- План лимитов: Настройки → Задать лимит. Бот пришлет шаблон месяца с категориями и суммами прошлого месяца.
- Цель: Цели → Добавить цель → `Отпуск 300000`.

## Расширение до версий 2.0 и 3.0

- Банковские выписки и Excel импорт лучше добавлять отдельным модулем импорта, который создает обычные `Transaction`.
- OCR чеков можно подключить перед `FinanceService.add_transaction`, чтобы распознанный чек проходил ту же категоризацию.
- LLM-категоризацию стоит расширять внутри `app/services/llm_service.py`, не меняя обработчики Telegram.
- Production-режим лучше разделить на два процесса: API и bot worker.
